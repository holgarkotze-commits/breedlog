Required:

- The normal individual animal export must include the visual family tree once, inside the main animal datasheet.
- Do not generate a second duplicate Family Tree / Pedigree Certificate page during normal animal export.
- Remove any code path that automatically appends the separate family tree page when the main datasheet already includes the tree.

Acceptance:

- PDF contains the visual family tree only once.
- No duplicate second family-tree certificate page is produced.
- Export UI no longer offers a confusing duplicate family-tree export option.

REQUIRED CHANGE 2 — MAIN INDIVIDUAL ANIMAL EXPORT MUST BE TRUE A4 PORTRAIT LAYOUT

The main individual animal datasheet must be a proper A4 portrait document.

Required:
- A4 portrait page size: 210mm x 297mm.
- Use correct print CSS:
  @page {
    size: A4 portrait;
    margin: 10mm;
  }
- The page content must fit inside the printable A4 area.
- The content must not cross page boundaries.
- The content must not be over-zoomed.
- The content must not be clipped.
- The content must not be tiny/shrunken with excessive unused white space.
- The content must fill the A4 page professionally from header to footer.
- Header must remain near top.
- Footer must remain near bottom.
- Main content must use the full available page height and width.

Important:
Do not confuse “PDF metadata says A4” with “the layout correctly fills A4.”
The exported PDF must visually look like a proper A4 page when rendered.
