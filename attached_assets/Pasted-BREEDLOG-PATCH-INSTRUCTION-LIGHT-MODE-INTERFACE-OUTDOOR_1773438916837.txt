BREEDLOG PATCH INSTRUCTION
LIGHT MODE INTERFACE + OUTDOOR VISIBILITY OPTIMISATION ONLY

IMPORTANT
This instruction concerns ONLY the implementation of a professional Light Mode interface and improved visibility for outdoor farm usage. No other system logic, database logic, herd classification logic, record systems, or UI structures must be altered. All existing functionality must remain unchanged. The objective is strictly visual optimisation and user readability.

------------------------------------------------------------

OBJECTIVE

Implement a professionally designed Light Mode interface for the BreedLog application that significantly improves readability, contrast, and usability when the application is used outdoors in direct sunlight on mobile devices.

The Light Mode must enhance visibility without changing the layout, structure, workflow, navigation, or behaviour of the existing application.

This change is strictly a visual theme enhancement.

------------------------------------------------------------

GENERAL REQUIREMENT

The Light Mode must be designed intentionally for high-visibility outdoor environments such as:

• livestock camps
• kraals
• loading ramps
• veld conditions
• open sunlight

The interface must remain clearly readable when viewed on a mobile phone in bright daylight.

The design must feel like a professional agricultural management tool, not a generic website theme.

------------------------------------------------------------

THEME SYSTEM

Introduce a Light Mode theme that works alongside the existing Dark Mode.

The system must support:

Light Mode  
Dark Mode  
System Auto

Default theme should be Light Mode.

The theme must apply globally to the entire application.

The theme switch must affect:

• backgrounds  
• text colors  
• tables  
• buttons  
• cards  
• dropdowns  
• filters  
• input fields  
• modals  
• icons  
• badges  
• headers  
• sidebars  

No interface element should remain styled incorrectly when Light Mode is active.

------------------------------------------------------------

DESIGN PRINCIPLE

The Light Mode must be designed with the following principles:

• maximum readability
• high contrast text
• non-glare backgrounds
• visible UI components
• professional agricultural aesthetic
• strong visual hierarchy
• easy scanning of herd data tables

Avoid washed-out colours and avoid low-contrast text.

------------------------------------------------------------

BACKGROUND COLOURS

Use an off-white base background to reduce glare in sunlight.

Base interface background:

#F6F8FA

Panel / card background:

#FFFFFF

This prevents eye strain compared to pure white.

------------------------------------------------------------

PRIMARY INTERFACE COLOUR

Use a strong agricultural-tech blue for primary actions.

Primary colour:

#1E4ED8

Use this colour for:

• active tabs
• primary buttons
• focus indicators
• links
• highlighted controls

------------------------------------------------------------

SECONDARY ACCENT COLOUR

Use a strong agricultural green for positive indicators.

Accent colour:

#15803D

Use this for:

• healthy animal indicators
• growth metrics
• positive statuses
• success confirmations

------------------------------------------------------------

WARNING AND CULL INDICATORS

Use strong red for critical warnings and culled animals.

Warning colour:

#DC2626

This colour must clearly stand out in tables and badges.

------------------------------------------------------------

TEXT COLOUR

Primary text colour:

#1E293B

Secondary text colour:

#334155

Text must always remain dark enough to be readable in sunlight.

Never use light grey text on light backgrounds.

------------------------------------------------------------

TABLE DESIGN IMPROVEMENT

Tables are one of the most frequently used components in BreedLog.

Tables must be optimised for visibility.

Requirements:

• strong header contrast
• alternating row shading
• clear row separation
• visible column titles
• strong hover highlight

Row colours:

Row 1: #FFFFFF  
Row 2: #F8FAFC

Header background:

#E2E8F0

Header text must be bold and clearly readable.

------------------------------------------------------------

INPUT FIELDS

All input fields must remain clearly visible in bright environments.

Fields must include:

• white background
• defined border
• visible focus state

Field border:

#CBD5F5

Focused border:

#1E4ED8

Placeholder text must remain readable.

------------------------------------------------------------

BUTTON DESIGN

Buttons must remain clearly visible outdoors.

Primary button:

background: #1E4ED8  
text: #FFFFFF

Secondary button:

background: #E2E8F0  
text: #1E293B

Danger button:

background: #DC2626  
text: #FFFFFF

Buttons must never visually blend into the background.

------------------------------------------------------------

TYPOGRAPHY

Font should remain clear and modern.

Preferred fonts:

Inter  
or  
Roboto

Minimum sizes:

Main UI text: 16px  
Table text: 15px  
Buttons: 16px  
Section headers: 20–26px  

Small unreadable fonts must not be used.

------------------------------------------------------------

BADGES AND STATUS INDICATORS

Badges must remain highly visible.

Recommended colour mapping:

Rams → Blue  
Ewes → Green  
Lambs → Amber  
Culled → Red

Badge colours must contrast clearly against the background.

------------------------------------------------------------

MOBILE TOUCH TARGETS

Because the application is used on farms while moving between camps:

• buttons must be easily tappable
• minimum touch size should be 44px
• controls must not be cramped

------------------------------------------------------------

THEME SWITCH LOCATION

Add theme control inside Settings or Admin.

Available options:

Light Mode  
Dark Mode  
System Auto

Default must be Light Mode.

User preference must persist after reload.

------------------------------------------------------------

DARK MODE

The existing Dark Mode must remain intact.

Light Mode must reuse the same component system and structure.

Only colour tokens and visual styling should change.

No layout changes are allowed.

------------------------------------------------------------

FINAL DESIGN EXPECTATION

When Light Mode is active the application must feel:

• professional
• structured
• modern
• highly readable
• suitable for outdoor agricultural work

The interface should resemble a professional livestock management instrument used by stud breeders and commercial livestock operations.

No structural UI changes must be introduced.

Only visual clarity, contrast, and outdoor usability improvements are required.

END OF INSTRUCTION
