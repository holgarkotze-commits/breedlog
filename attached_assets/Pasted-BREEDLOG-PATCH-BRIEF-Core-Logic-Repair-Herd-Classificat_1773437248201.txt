BREEDLOG PATCH BRIEF
Core Logic Repair, Herd Classification Correction, Count Restoration, Records Logging Fix
Objective

Repair BreedLog’s herd-classification logic, count logic, section consistency, and records/export logging so that all herd sections, homepage totals, records totals, exports, and animal record tabs work from one correct source of truth.

This patch must not introduce workaround logic or cosmetic-only fixes. The fix must be structural and reliable across all screens, exports, and sync events.

1. Herd classification logic must be rebuilt
Required rule

Animal section placement must no longer depend on manually trusted category labels or inconsistent UI state.

The current herd section for each animal must be derived from:

date of birth

sex

cull status

Required section derivation logic

Apply this exact logic in this order:

If animal is marked as Cull, animal belongs to Culled

Else if animal age is under 8 months, animal belongs to Lambs

Else if sex is Ram, animal belongs to Rams

Else if sex is Ewe, animal belongs to Ewes

This derived section logic must become the single logic source for:

homepage counts

herd section counts

herd tab badge counts

herd table contents

exports

reports

records where applicable

Do not allow different screens to calculate section membership differently.

2. New animal entry form must be corrected
Current problem

The app currently allows incorrect logic by treating “Lamb” as a sex/category input.

Required fix

When creating a new animal entry, the sex field must only allow:

Ram

Ewe

Remove Lamb as an available selection from the sex input.

Required behavior after entry

When a new animal is entered:

user chooses Ram or Ewe as sex

user enters date of birth

if age is under 8 months, the animal must appear in Lambs

once system-calculated age reaches 8 months, the animal must automatically move to:

Rams if sex = Ram

Ewes if sex = Ewe

if the animal is marked Cull at any time, the animal must appear in Culled instead of Rams/Ewes/Lambs

This movement must be automatic and date-driven, not dependent on manual recategorization.

3. My Herd structure must be updated to 5 sections
Required herd sections

The “My Herd” area must now contain exactly these five sections:

Total Herd

Rams

Ewes

Lambs

Culled

Insert the new Culled section below Lambs.

4. Culled section must match Total Herd section behavior
Required behavior

The new Culled section must have exactly the same:

button features

dropdown lists

column names

animal info layout

search behavior

filter options

class options

display view options

as the existing Total Herd section.

This must not be a reduced or simplified section. It must be functionally equivalent to Total Herd, with only the base dataset changed to culled animals.

5. All 5 herd sections must use the same filter system
Required rule

All five sections must use exactly the same filter framework and filter options as Total Herd.

This applies to:

Total Herd

Rams

Ewes

Lambs

Culled

Required parity

All 5 sections must now include exactly the same:

filter options

class options

Tag ID search bar

display view options

search and dropdown behavior

Only the base dataset changes by section. The filter system must be shared and consistent.

6. Count logic across the app must be rebuilt from one source of truth
Current problem

Homepage counts, herd section counts, and displayed record counts do not match. Example already identified:

homepage total herd = 15

homepage ewes = 4

homepage lambs = 6

homepage rams = 4

But 4 + 6 + 4 = 14

Also:

homepage ewe count differs from ewe section count

same mismatch exists for rams and lambs

current count logic is unreliable across the app

Required fix

Create one centralized herd summary calculation source and ensure all screens read from it.

Required outputs

The system must generate consistent counts for:

Total Herd

Rams

Ewes

Lambs

Culled

Required consistency

The following must always match:

homepage card count

section tab count

section badge count

actual displayed section records before user-applied filters

export totals where applicable

No screen may maintain its own disconnected count logic.

7. Define and enforce Total Herd calculation
Required decision

Implement Total Herd as one clearly defined count and use that same definition everywhere.

Recommended implementation:

Total Herd = Rams + Ewes + Lambs + Culled

If the current product owner later wants Total Herd to exclude culled animals, then that change must be made globally and consistently everywhere. Do not leave mixed definitions across screens.

For this patch, implement one definition only and make sure all pages obey it.

8. Section counts must match actual section contents
Required rule

Each herd section count must equal the number of animals actually classified into that section.

This applies to:

Rams section

Ewes section

Lambs section

Culled section

Total Herd section

If a section shows 7 animals in the table, the section count must also be 7.
If homepage says 4 ewes, then the Ewes section must also resolve to 4 ewes from the same source if no additional filters are active.

The count shown must always reflect the same live classified dataset used to render the section.

9. Automatic lamb transition at 8 months must be implemented
Required logic

A lamb remains in the Lambs section from birth until it reaches exactly 8 months old based on date of birth.

Once the animal reaches 8 months:

Ram sex -> move to Rams

Ewe sex -> move to Ewes

If Cull status is active -> move to Culled instead

This transition must happen automatically from system calculation and must remain correct after refresh, sync, export, and reload.

Do not require the user to manually move animals.

10. Cull status must override normal section placement
Required logic

Cull status must override all normal classification placement.

That means:

if an animal is marked Cull, it must appear in Culled

it must not remain in Lambs, Rams, or Ewes once Cull is active

this must apply whether the animal is below or above 8 months

Cull logic must be respected everywhere:

herd sections

counts

exports

reports

filters

11. Export and records logging system must be fixed
Current problem

Exported documents are not being counted in Records. Counts remain at 0 even after multiple exports.

Examples already identified:

herd exports created

individual exports created

health records exported

records counts still show 0

Required fix

Every export action must create a persisted record entry in the records/logging system.

Each export record must store at minimum:

record type

export category

affected quantity

related animal ID if applicable

timestamp

document reference or filename

sync status if used

any existing required metadata already used by the app

Records screen must count actual persisted records

Do not infer counts from UI state.
Do not calculate counts from temporary in-memory arrays only.
Do not leave export actions unlogged.

The Records area must correctly count and display at minimum:

herd exports

individual exports

health exports

other existing record/export categories already supported in the app

12. Quantity fields must be stored for records
Required rule

All records and export actions must store quantity values properly.

Where batch or document actions occur, the system must record quantity explicitly.

Examples:

number of animals included in herd export

number of animals included in selected export

number of health records exported

number of records affected in a batch event

Do not rely only on text labels. Quantity must be stored numerically where relevant.

13. Health tab in animal records must be fixed
Current problem

Health tab is not shown at animal records, while health-related actions and exports exist elsewhere.

Required fix

Restore and correctly display the Health tab in animal records where health records are supported.

Required behavior

The Health tab must:

render correctly in animal records

show associated health information if available

integrate correctly with record counts and exports

not disappear due to broken conditional rendering, missing registration, or schema mismatch

If health records exist, the tab must be visible and functional.

14. Records tab counts must be corrected across all entries
Required fix

All counts shown under the Records tab must reflect actual saved records.

This includes correcting counts for:

herd exports

individual animal exports

health records / health exports

any other record/export categories currently implemented

Counts must update correctly after:

new export

record creation

page refresh

sync

app reload

No record category should remain stuck at zero after valid actions have been performed.

15. Sync-safe logic is required
Required rule

Count logic, section logic, and records logic must survive sync and reload correctly.

After sync, refresh, or app restart, the following must still remain correct:

animal classification by section

homepage totals

herd section counts

records counts

export counts

health tab visibility

cull placement

lamb automatic age transition

Do not implement fixes that only work until the next refresh.

16. Technical architecture requirement
Required implementation direction

Refactor so the app uses shared centralized logic instead of multiple disconnected count calculations.

The system must include:

A. Shared animal classification logic

One function/module/service that determines each animal’s current derived section from:

DOB

sex

cull status

B. Shared herd summary logic

One function/module/service that calculates:

total herd count

ram count

ewe count

lamb count

culled count

C. Shared records summary logic

One function/module/service that calculates:

record totals by type

export totals by type

health totals where relevant

D. Shared herd-list rendering base

All 5 herd sections must use the same list/table/filter component logic with different base datasets only.

Do not leave legacy per-screen calculation logic in place if it causes drift or mismatch.

sex stays Ram or Ewe, never Lamb

Rams

full herd-section functionality identical to Total Herd
