CRITICAL CHANGE: REMOVE “SIGN IN WITH REPLIT”. BreedLog users must NOT need a Replit account.

Goal:
Implement a controlled beta access system that works with NO Replit authentication. Users should install and use BreedLog without creating any external accounts.

REQUIREMENTS (MUST):

1) Remove Replit Auth Completely
- Remove any “Sign in with Replit” button/flow.
- Remove reliance on Replit user identity for admin or testers.

2) Access Code + Device Binding (No Login)
- On first launch, show “Enter Access Code” screen.
- User enters code -> app activates without any account creation.
- Generate a secure “deviceId” on the device (random UUID) and store locally.
- When code is first used:
  - bind code to deviceId (max 1 device only)
  - activate access for that device
- If same code is used on a second device -> BLOCK and show “Code already used.”

3) Beta Rules (Hard Requirements)
- Expiry: 30 days default
- Max uses: 1 device only
- Instant revoke: admin can revoke and kill access
- Offline grace: app works offline for max 7 days after last successful validation, then locks until online check
- Only 10 active testers allowed total (system must enforce this)

4) Admin Panel WITHOUT Replit Login
- Admin access must be protected without Replit auth:
  Use ONE of these approaches (choose simplest):
  A) Admin secret PIN/passphrase stored as environment secret (ADMIN_PIN) to access /admin
  OR
  B) Admin “master code” that unlocks admin panel on device and stores local admin flag
- Admin panel must allow:
  - Create invite codes (expiry, max uses, note)
  - Revoke invite codes instantly
  - See active tester count (must not exceed 10)
  - See code status + bound deviceId + last validation time

5) Data Isolation (Still Mandatory)
- All farm/herd/animal/images/records must be scoped to a local userId linked to deviceId.
- One tester must NEVER see another tester’s data.

6) Testing (Must Confirm)
- New user installs -> no Replit sign-up -> enters code -> works.
- Second device tries same code -> blocked.
- Offline works up to 7 days -> locks after grace until online check.
- Revoke code -> device loses access on next online check.
- More than 10 active testers -> new activations blocked.
- Users do not share farm data.
