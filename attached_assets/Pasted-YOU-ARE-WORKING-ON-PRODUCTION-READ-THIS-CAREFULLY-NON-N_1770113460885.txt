Admin must be able to:

- MAX_TESTERS must be editable in Admin UI (default 50).

5) ACTIVATION MUST BE TOKEN-BASED (NO SESSION COOKIE RELIANCE)
- Session cookie-based flows are unreliable on mobile browsers.
- Activation flow must:
  a) Ensure device gets a unique deviceId
  b) Validate invite code
  c) Create activation record linking deviceId + code
  d) Issue device token
  e) Store token in localStorage (or IndexedDB) and attach on every API call in headers
- App must not randomly fail after refresh/reopen.
- All device-protected endpoints must accept the token and not depend on session.

6) SERVICE WORKER + CACHE HARD RULES (NO STALE ADMIN DATA)
- Service worker MUST BYPASS entirely:
  /admin/*
  /api/admin/*
  /api/beta/*
  /api/device/*
- Backend must return no-cache headers for these endpoints:
  Cache-Control: no-store, no-cache, must-revalidate, proxy-revalidate
  Pragma: no-cache
  Expires: 0
- Client fetch must also send cache-busting where needed.
- Admin “Hard Refresh” button must append a timestamp param and refetch.

7) OFFLINE/ONLINE BEHAVIOUR
- After activation: user can use the app offline.
- During activation: if offline, show a clear message “You must be online to activate”.
- No random “device not authenticated” after a successful activation.
