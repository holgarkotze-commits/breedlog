Fix BreedLog mobile UI polish and dark-mode readability across the whole app.

Current status:
The app is loading and the AI assistant connects, but the mobile UI is not professionally optimized yet.

Observed issues from phone screenshots:

1. BreedLog Assistant modal is not mobile-safe

* The AI response/error box exceeds the phone screen width.
* Text overflows horizontally.
* The send/input area does not fit cleanly on mobile.
* The send button is too dark and unclear.
* Long backend/provider errors are displayed raw and destroy the mobile layout.

2. Dark-mode button text is unreadable in multiple places
   Examples:

* Add Animal button
* New Group button
* Record Event button
* Health Records button
* Activate Access button
* AI send button
* other blue/dark gradient buttons

Rule:
In dark mode, every button must have clear readable text.
Primary rule: button text must be white or high-contrast, never dark-on-dark.

3. Refresh/sync floating button overlaps the BreedLog logo

* On mobile, the refresh/sync widget sits on top of the logo.
* It stays over the logo while scrolling.
* It looks unprofessional and must be repositioned.
* It must remain usable but must not cover branding, headings, cards, or important page actions.

Required fixes:

A. Mobile assistant layout
Make the BreedLog Assistant fully mobile responsive:

* modal width must never exceed viewport width
* no horizontal page overflow
* use max-width: calc(100vw - safe padding)
* long text must wrap correctly
* response area must scroll vertically inside the modal when needed
* input row must fit inside screen
* send button must remain visible and tappable
* close button must be properly aligned and tappable
* suggested question chips must wrap/scroll cleanly without pushing layout wider than screen

B. AI error handling
Do not show raw provider JSON/errors to users.
For 429 / RESOURCE_EXHAUSTED / quota errors, show a clean message like:
“AI quota is temporarily exhausted. Please try again later.”
Keep the detailed provider error only in server logs/dev logs.
The user-facing assistant card must stay compact and readable.

C. Dark-mode button contrast system
Audit all button styles across the app.
Fix the theme globally so in dark mode:

* dark/blue/gold/gradient buttons use white readable text
* disabled buttons are still readable but visibly disabled
* icons remain visible
* hover/active/focus states remain accessible
* no button has black/dark text on a dark blue gradient background

Do not patch only one button. Fix the shared button classes/components/theme tokens so the rule applies everywhere.

D. Refresh/sync floating widget placement
Reposition the refresh/sync indicator on mobile so it does not overlap the BreedLog logo.
Options:

* move it below the logo area
* dock it neatly to the right with safe top spacing
* or integrate it into a header/status area

Requirements:

* must not cover the logo
* must not cover page headings
* must not cover action buttons
* must respect mobile safe-area spacing
* must look intentional and polished
* must work on all main pages: Home, Herd, Data, Breeding, More, Health, AI assistant overlay

E. Mobile alignment audit
Check and fix mobile alignment/spacing on:

* Herd page
* Breeding page
* Health page
* Data page
* More page
* Access code page
* Admin/access-related screens if visible on mobile
* BreedLog Assistant overlay

F. Tests/checks
Add or update UI tests where practical for:

* no horizontal overflow on mobile viewport
* assistant modal fits mobile viewport
* raw AI provider JSON is not rendered to user
* dark mode primary/action buttons have readable text styling
* floating refresh/sync widget does not overlap logo/header area

Run and report:

* files changed
* root cause of mobile overflow
* root cause of dark button text issue
* root cause of refresh widget overlap
* npm run check result
* npm run build result
* any UI/responsive tests run

Acceptance gate:
This is not complete until the app looks professional on mobile dark mode, the AI assistant fits fully inside the phone screen, no raw provider JSON appears to the user, all dark-mode buttons have readable white/high-contrast text, and the refresh/sync widget no longer overlaps the BreedLog logo or main content.
