# BreedLog Evidence Pack

Folders are scoped by phase area: baseline, architecture, data-isolation, migrations, authentication, billing, entitlements, downgrade-restoration, backup-reset-restore, exports, pwa, android, windows, security-privacy, legal-tax, monitoring-support, and final-release.

Allowed file types are `.md`, `.txt`, `.log`, `.json`, `.csv`, and small redacted `.png` screenshots. Use names like `YYYY-MM-DD-command-or-check.log`. Every evidence file must include command/source, working directory, environment, commit/branch when available, result as PASS/FAIL/BLOCKED, and redaction notes. Do not commit secrets, tokens, private keys, signing material, private farm data, large binaries, traces, or unredacted screenshots.
